# Variant: Strategy Memo

**Mode:** Client Deliverable.

## When to use

The user needs a written deliverable -- a strategy memo, board paper, decision document, or executive brief. The output is a document, not a conversation. Because the user has explicitly asked for a deliverable, missing details get *named* as `[A]` assumptions in the memo itself rather than blocking the draft.

## Structure: Pyramid Principle + SCQA

Every strategy memo follows the same logic:

1. **Situation** -- shared context the reader already knows (1-2 sentences).
2. **Complication** -- what changed, what's at stake, why this matters now.
3. **Question** -- the specific question this memo answers.
4. **Answer** -- the recommendation, stated upfront (1 sentence).

Then the body supports the answer with grouped arguments, each backed by evidence.

## Executive summary template

```
EXECUTIVE SUMMARY

[Situation]: [Company/team] currently [status quo context].
[Complication]: However, [what changed / what's at risk / trigger event].
[Question]: The question is [specific decision question].
[Answer]: We recommend [specific action] because [1-sentence rationale].

This will require [investment/effort] and deliver [expected outcome] by [date].
The main risk is [top risk], mitigated by [mitigation].
```

Keep the executive summary to 6-8 lines. If it takes more, you haven't sharpened the thinking enough.

## 5-page strategy memo outline

### Page 1: Executive summary + recommendation
- SCQA (4-6 lines)
- Recommendation (1 sentence, bolded)
- Key numbers: investment required, expected return, timeline
- Decision requested: what you need from the reader

### Page 2: Context and problem definition
- Market/competitive backdrop (only what's relevant -- not a full industry overview)
- The specific problem or opportunity, with data
- Why now -- the trigger or burning platform
- Scope: what this memo covers and what it does not

### Page 3: Analysis and options
- 2-3 options evaluated (always include "do nothing" as a baseline)
- For each: description, pros, cons, financial impact, feasibility
- Evaluation criteria (stated explicitly -- what dimensions are we judging on?)
- Comparison table or matrix

### Page 4: Recommended path + implementation
- Recommended option with rationale (connect back to evaluation criteria)
- Implementation roadmap: phases, milestones, owners
- Resource requirements: budget, headcount, timeline
- Dependencies and sequencing

### Page 5: Risks + next steps
- Top 3-5 risks with likelihood, impact, and mitigation
- Monitoring plan: how we'll know if it's working (leading indicators)
- Immediate next actions (each with owner and deadline)
- Decision deadline and escalation path

## Slide outline format

When the user needs a deck outline instead of a prose memo:

```
Slide 1: Title + recommendation (1 sentence)
Slide 2: Situation -- market context (1 chart + 3 bullets)
Slide 3: Complication -- the problem (data visualization)
Slide 4: Options considered (comparison table)
Slide 5: Recommendation detail (what, who, when, how much)
Slide 6: Financial impact (P&L bridge or waterfall)
Slide 7: Implementation roadmap (Gantt or phased milestones)
Slide 8: Risks and mitigations (risk matrix)
Slide 9: Next steps and ask (decision + actions)
```

Each slide should have: one headline (a sentence, not a label), one visual, and 2-4 supporting bullets.

**Headline rule:** Slide headlines are assertions, not topics. "Revenue grew 23% driven by enterprise segment" -- not "Revenue overview."

## Writing discipline

- **One idea per paragraph.** If a paragraph makes two points, split it.
- **Lead with the conclusion.** Every section starts with its bottom line, then supports it.
- **Use active voice.** "We recommend X" not "It is recommended that X."
- **Numbers need context.** "$5M" means nothing without "vs $3.2M last year" or "representing 12% of revenue."
- **Cut ruthlessly.** If a sentence doesn't advance the argument, delete it. Strategy memos are not reports -- they drive decisions.

## Common mistakes in strategy memos

- Burying the recommendation on page 4 (put it on page 1, sentence 1).
- "Boil the ocean" context sections that review the entire industry before getting to the point.
- Options that aren't real options (one great option + two obviously bad ones = no real choice).
- Missing the "do nothing" baseline -- every recommendation needs to be compared against inaction.
- Risks without mitigations (identifying risk is easy; showing you've thought about response is the hard part).
